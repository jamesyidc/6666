## 🎨 界面功能说明

### 1. 状态栏（顶部）
显示实时的市场数据：
- **运算时间** - 数据采集时间
- **急涨** - 急涨币种数量（绿色）
- **急跌** - 急跌币种数量（红色）
- **状态** - 市场状态（震荡无序/趋势明确等）
- **比值** - 急涨/急跌比值
- **差值** - 急涨-急跌
- **计次** - 市场波动计次
- **24h涨幅≥10%** - 大涨币种数量
- **24h跌幅≤-10%** - 大跌币种数量

### 2. 日期时间选择器
- **选择日期** - 可以选择任意历史日期
- **时间** - 选择具体时间（时:分）
- **查询按钮** - 🔍 查询指定时间的数据
- **最新数据按钮** - 📊 快速加载最新数据

### 3. 币种数据表格
完整显示所有币种信息：

| 列名 | 说明 |
|------|------|
| 序号 | 币种排序 |
| 币名 | 币种符号（如BTC、ETH） |
| 涨跌 | 当前涨跌幅（红色为涨，绿色为跌） |
| 急涨 | 急涨标记（0或1） |
| 急跌 | 急跌标记（0或1） |
| 更新时间 | 数据更新时间 |
| 历史高点 | 历史最高价格 |
| 高点时间 | 达到最高价的时间 |
| 跌幅 | 从高点下跌的幅度 |
| 24h% | 24小时涨跌幅 |
| 排行 | 市值排名 |
| 当前价格 | 实时价格 |
| 最高占比 | 最高价格占比 |
| 最低占比 | 最低价格占比 |
| **优先级** | **等级1-6（颜色标记）** |

### 4. 趋势图
使用ECharts绘制的专业图表，包含4条曲线：
- **急涨**（红色实线）- 急涨币种数量变化
- **急跌**（绿色实线）- 急跌币种数量变化
- **差值**（橙色虚线）- 急涨-急跌的差值
- **计次**（蓝色实线）- 市场波动计次（右侧Y轴）

## 📖 使用教程

### 查看最新数据
1. 打开网页
2. 点击 **📊 最新数据** 按钮
3. 自动显示最新的币种数据和趋势图

### 查询历史数据
1. 点击 **选择日期** 输入框，选择日期（如：2025-12-06）
2. 点击 **时间** 输入框，选择时间（如：13:42）
3. 点击 **🔍 查询** 按钮
4. 显示指定时间点的数据

### 分析优先级
1. 在表格中找到 **优先级** 列
2. 关注红色（等级1）和橙红色（等级2）的币种
3. 这些是根据最高占比和最低占比计算出的高优先级币种

### 查看趋势
1. 向下滚动到趋势图
2. 查看急涨/急跌/差值/计次的历史变化
3. 红色线上升表示急涨增多，绿色线上升表示急跌增多
4. 橙色虚线是差值，蓝色线是计次

## 💡 使用技巧

### 快速查找高优先级币种
1. 在表格中，优先级等级1-3的币种会有明显的颜色标记
2. 直接查看表格最后一列的颜色即可

### 判断市场状态
1. 查看状态栏的"状态"字段
2. 查看急涨和急跌的数量对比
3. 查看差值是正数（偏多）还是负数（偏空）

### 对比不同时间点
1. 先查询一个时间点，记录数据
2. 再查询另一个时间点
3. 对比急涨、急跌、计次等指标的变化

## 📊 数据来源

- 所有数据从 `crypto_data.db` SQLite数据库读取
- 数据通过 `collect_and_store.py` 脚本采集
- 建议每10分钟运行一次采集脚本以获取最新数据

## 📝 注意事项

## ❓ 常见问题

**Q: 为什么图表没有数据？**  
A: 需要先运行 `collect_and_store.py` 采集足够的历史数据。

**Q: 优先级是怎么计算的？**  
A: 根据最高占比和最低占比自动计算，具体规则见上方优先级颜色说明表。

**Q: 可以查询未来的时间吗？**  
A: 不可以，只能查询已经采集过的历史数据。

**Q: 数据多久更新一次？**  
A: 取决于运行 `collect_and_store.py` 的频率，建议每10分钟运行一次。
